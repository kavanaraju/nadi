Code & References

Notebooks

The full analysis is implemented across four Jupyter notebooks, available in the repository.

1. Data acquisition

View notebook →

Downloads LiDAR tiles from USGS, processes building footprints, extracts the pedestrian network from OSM, and loads SEPTA transit stops.

Outputs: buildings_with_heights.geojson, edges.geojson, septa_stops.geojson

2. Shade calculation

View notebook →

Rasterizes LiDAR for buildings and trees, models solar position across 8 scenarios, calculates geometric shadows, buffers street segments, and computes shade scores.

Outputs: buildings_heights.tif, tree_heights.tif, network_edges_with_shade.geojson

Runtime: ~2 hours

3. Routing analysis

View notebook →

Constructs the network graph, implements Dijkstra’s algorithm with shade-weighted cost function, calculates test routes, and generates comparison metrics.

Runtime: ~15 minutes

4. Visualization

View notebook →

Produces static route comparison maps, shade heatmaps, and summary charts.

Runtime: ~10 minutes

Data dictionary

network_edges_with_shade.geojson

Column Type Description
u int Start node ID
v int End node ID
key int Edge key (for multi-edges)
osmid int/list OpenStreetMap ID(s)
name str Street name
highway str OSM highway type
length float Segment length (feet)
geometry LineString Segment geometry (WGS84)
building_shadow_summer_midday float Building shade score (0-1)
tree_shadow_summer_midday float Tree shade score (0-1)
shade_summer_midday float Combined shade score (0-1)
(repeated for all 8 scenarios)

Shade score interpretation: 0.0 = full sun, 0.5 = half the pedestrian zone shaded, 1.0 = fully shaded.

Reproducing the analysis

git clone https://github.com/kavanaraju/nadi.git
cd nadi

conda env create -f environment.yml
conda activate shade-routing

jupyter notebook
# Run notebooks in order: 01 → 02 → 03 → 04

quarto render

Total runtime approximately 5 hours, dominated by the shade calculation step.
